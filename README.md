## ✨ 핵심 기능 (Key Features)

### 1. 📊 실시간 관제 대시보드

- **자동 갱신 & 무한 스크롤**: 5초마다 백그라운드에서 최신 트랜잭션 데이터를 동기화(Polling)하며, `IntersectionObserver`를 활용한 무한 스크롤로 브라우저 성능 저하 없이 대량의 데이터를 시각화합니다.
- **검색 및 다중 필터링**: Trace ID나 기관명으로 즉시 검색하고, 상태별(정상, 장애, 지연) 필터링 및 프로그레스 바 차트를 실시간으로 제공합니다.

### 2. 🚨 장애/미처리 전용 대기열 (DLQ)

- **원인별 격리 & 필터링**: 500(서버 에러), 504(타임아웃) 등 응답 코드별로 실패/지연 건을 모아 집중 분석합니다.
- **일괄 재처리 (Bulk Retry)**: 장애 건을 다중 선택하여 비동기(`Promise.all`)로 일괄 재전송을 요청하거나, 영구 실패 건을 '처리 포기'하여 대기열에서 소거합니다.

### 3. ⚙️ 제휴사 및 인터페이스 정책 (Config) 관리

- **도메인 분리 & 컴포넌트화**: 담당자 연락처를 관리하는 '제휴사'와 라우팅/타임아웃을 관리하는 '인터페이스 정책'을 분리하고, 순수 Vanilla JS 컴포넌트로 모듈화하여 관리합니다.
- **참조 무결성 UX**: 제휴사가 추가/수정되면 인터페이스 정책 등록 폼의 드롭다운 목록이 실시간으로 동기화됩니다.

### 4. 📝 운영 작업 이력 (Audit Log)

- **컴플라이언스 추적**: 수동 재처리, 처리 포기, 정책 등록/수정/삭제 등 시스템 상태를 변화시키는 모든 운영자의 액션을 백엔드 DB에 자동 기록하여 투명하게 조회합니다.

### 5. 🧪 능동형 장애 시뮬레이터

- **임의 시뮬레이션 자동 발생**: 등록된 제휴사/정책을 기반으로 정상(60%), 장애(20%), 지연(20%) 상태의 트랜잭션을 2초 간격으로 연속 5회 자동 생성하여 모니터링 반응을 테스트할 수 있습니다.
- **수동 에러 주입**: HTTP 상태 코드 및 JSON 페이로드 전문을 직접 작성하여 세밀한 엣지 케이스를 재현합니다.

### 6. 🛡️ 엣지 케이스 방어 및 편의 기능

- **위험 알림 (Alerting)**: 미처리 장애 건수가 3건 이상 누적되면 화면에 즉각적으로 경고 팝업을 띄웁니다.
- **데이터 마스킹 & CSV 다운로드**: 모달창에서 민감 정보를 `**`로 마스킹 처리하며, 조회된 데이터 목록을 즉시 엑셀(CSV) 형태의 파일로 추출할 수 있습니다.
- **동적 URL 라우팅**: 로컬 및 운영 배포 환경을 프론트엔드가 자동 감지하여 백엔드 API 주소를 유동적으로 변경합니다.

## 🛠️ 기술 스택 & 아키텍처 (Tech Stack & Architecture)

### Frontend

- **Framework:** Vanilla JS (의존성 패키지 없이 순수 웹 표준 기술)
- **Architecture:** Component 기반 설계 (`view-partner.js`, `view-policy.js`) 및 SPA 라우팅
- **Styling:** CSS3 (Flexbox/Grid 기반 완벽한 반응형 대응)

### Backend

- **Framework:** Java 17, Spring Boot 3.x
- **Database:** H2 In-memory DB / Spring Data JPA (Hibernate)
- **Architecture:** REST API, Layered Architecture (Controller - Service - Repository)
- **Security:** `.env` 템플릿화를 통한 DB 접속 정보 등 환경변수 분리

## 실행 방법 (Getting Started)

본 프로젝트는 프론트엔드와 백엔드가 철저히 분리된 모노레포 구조를 가집니다.

### 💻 Frontend 실행

별도의 빌드 과정 없이 정적 파일만으로 구동됩니다.

1. `front/` 폴더로 이동합니다.
2. `index.html` 파일을 브라우저로 드래그하여 열거나, VS Code의 **Live Server**로 실행합니다.

### ⚙️ Backend 실행

1. `back/` 폴더로 이동하여 IDE(IntelliJ 등)로 프로젝트를 엽니다.
2. 프로젝트 최상단의 `.env.template`을 복사하여 `.env`로 이름을 변경합니다. (기본 H2 설정 사용 시 생략 가능)
3. `BackApplication.java`를 실행하거나 터미널에서 `./gradlew bootRun`을 입력하여 `localhost:8080` 포트로 기동합니다.
4. 기동 시 5개의 제휴사와 6개의 인터페이스 정책이 DB에 자동 세팅됩니다.
