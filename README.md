# 🛡️ Noa Insure: 통합 인터페이스 관제 플랫폼

> **"외부 기관 통신 장애의 조기 인지 및 신속한 복구를 위한 중앙화된 인터페이스 관제 플랫폼"**  
> 다수의 외부 기관(금감원, 제휴사 등)과 내부 핵심 시스템 간의 데이터 통신 상태를 단일 화면에서 제어하고, 장애 발생 시 운영자가 즉각적으로 개입할 수 있는 웹 기반 통합 관리 플랫폼입니다.

---

## 🎯 1. 문제 정의 및 기획 배경

- **해결하려는 문제:** 기존 분절된 시스템 환경에서는 외부 기관과의 데이터 전송 장애 발생 시, 운영자가 즉각적으로 인지하지 못해 **'조용한 실패(Silent Failure)'**로 이어지는 치명적인 문제가 있습니다.
- **비즈니스 임팩트 (선택 근거):** 금융 환경에서 원가/회계 데이터나 고객 실적 데이터의 전송 누락은 곧 컴플라이언스 위반 및 금융 사고로 직결됩니다. 따라서 장애를 즉시 시각화하고, 복잡한 서버 접근 없이 화면에서 **'즉시 재처리(Retry)'** 할 수 있는 관제 시스템이 필수적입니다.

## 👥 2. 대상 사용자 및 시스템 목표

- **대상 사용자:** 보험사 내부 IT 인프라 운영자 및 시스템 관리자
- **시스템 목표:** 장애 인지 소요 시간 최소화 및 운영 리소스(서버 로그 직접 확인 등) 낭비 방지

---

## ✨ 3. 주요 기능 (Core Features)

- **📊 실시간 관제 대시보드:** 전체 인터페이스의 성공/실패 상태, 트래픽 현황, 지연 상태를 한눈에 파악할 수 있는 시각화 대시보드.
- **📋 인터페이스 운영 리스트:** 대상 프로토콜(REST API, SFTP, MQ, SOAP, Batch 등)과 기관별 통신 상태를 확인하는 통합 그리드.
- **🚨 장애/미처리 전용 대기열 (DLQ):** 실패한 통신 건들만 모아서 에러 사유별로 필터링하고 집중 관리하는 전용 뷰 제공.
- **🔄 원클릭 장애 재처리 (수동 & 일괄 Retry):** 통신 실패(Error) 처리된 데이터를 시스템 관리자가 화면에서 즉각적으로 재전송하거나 여러 건을 선택해 일괄 처리.
- **🧪 장애 시뮬레이터 (테스트 모듈):** 실제 환경 도입 전, 특정 인터페이스에 고의적인 타임아웃이나 서버 에러(JSON 페이로드 포함)를 발생시켜 관제 시스템의 반응을 테스트하는 기능.

## 🔒 4. 보안 및 추가 기능 (Additional Features)

- **📝 상세 로그 및 페이로드(Payload) 뷰어:** 통신 실패 원인 분석을 위해 송수신된 JSON 데이터 전문과 HTTP Status Code를 모달창으로 제공.
- **🛡️ 보안 마스킹 (Data Masking):** 페이로드 조회 시 고객 이름, 계좌번호 등 민감 정보에 대해 정규식을 활용한 `**` 마스킹 처리 적용 (운영자 화면 보안).
- **🔗 Trace ID 추적:** 트랜잭션마다 고유 ID를 부여하여 내부 시스템부터 외부 기관까지의 전체 데이터 흐름 연계 식별.

---

## 🏃 5. 사용자 시나리오 (User Scenario)

1. **[모니터링]** 운영자가 관제 대시보드에 접속하여 전체 시스템이 '정상' 가동 중임을 확인한다.
2. **[장애 발생]** 금감원 원가 보고서 전송 인터페이스에서 외부 서버 무응답으로 인한 '500 Error(Timeout)'가 발생한다. _(장애 시뮬레이터로 완벽한 재현 가능)_
3. **[장애 인지]** 대시보드 시각화 차트에 붉은색 경고 비율이 늘어나고, 운영 리스트에서 해당 인터페이스 상태가 '장애'로 즉각 변경된다.
4. **[원인 분석]** 운영자가 해당 건의 Trace ID를 클릭하여 마스킹된 데이터 전문(Payload)과 'Connection Timeout' 에러 메시지를 안전하게 확인한다.
5. **[조치 및 복구]** 원인 파악 후, 운영자가 **[재처리]** 버튼을 클릭한다. 시스템이 데이터를 재전송하고 상태가 다시 '정상'으로 복구된다. _(다수 장애 발생 시 DLQ 메뉴에서 일괄 재처리 수행)_

```text

 ┣ 📂 .gemini/      # AI 어시스턴트 코딩 컨벤션 및 개발 마인드셋 문서
 ┣ 📂 docs/         # 시스템 연동 기록, 신규 기능 명세 등 개발 히스토리
 ┣ 📂 front/        # 순수 Vanilla JS, HTML, CSS 기반의 반응형 SPA 프론트엔드
 ┗ 📂 back/         # 백엔드 API 및 서버 비즈니스 로직 (구축 예정)
```
