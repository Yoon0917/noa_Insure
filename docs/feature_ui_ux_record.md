# 신규 기능 구현 기록: 반응형 UI 개선 및 무한 스크롤(Infinite Scroll) 적용

## 구현 일자: 2026-04-26

### 📌 기획 배경 및 목적

모바일 및 작은 창(Window)에서 버튼과 폼 요소가 화면 밖으로 밀려나는 레이아웃 붕괴 현상이 발견되었습니다. 또한 데이터가 수천 건 이상 쌓일 경우 한 번에 DOM을 렌더링하면 브라우저 성능이 크게 저하되는 병목 현상이 우려되어 렌더링 최적화가 필요했습니다.

### ⚙️ 핵심 구현 사항

1. **무한 스크롤 (Infinite Scrolling) 성능 최적화**
   - 관제 대시보드와 장애 대기열(DLQ) 테이블에 브라우저 내장 API인 `IntersectionObserver`를 도입했습니다.
   - 전체 데이터를 메모리에 유지하되, 사용자가 스크롤하여 테이블 하단 도달 시 20건(Chunk)씩 추가로 DOM에 이어 붙이는(`append=true`) 방식으로 렌더링 속도와 체감 성능을 극대화했습니다.

2. **테이블 및 모달 반응형 스크롤 (Overflow 방어)**
   - 데이터 그리드 래퍼(`table-responsive`)에 `overflow-x: auto`를 적용하여, 화면 너비가 좁아져도 전체 레이아웃이 함께 넓어지지 않고 표 내부에서만 스와이프가 가능하도록 조치했습니다.
   - 입력 폼이 긴 모달창(Modal) 몸체 영역에 `max-height: 75vh`와 `overflow-y: auto`를 부여하여 화면이 작은 기기에서도 '등록 완료' 액션 버튼이 잘리지 않고 표시되도록 개선했습니다.

3. **입력 폼 가변 그리드(Flex Wrap) 적용**
   - 일렬로 나열되어 있던 입력 폼들에 `flex-wrap: wrap`과 `flex: 1 1 200px` 등의 가변 폭(Flexbox) 속성을 적용했습니다.
   - 덕분에 데스크탑에서는 가로로 넓게 배치되고, 모바일에서는 여백에 따라 자연스럽게 세로로 나열되도록 반응형을 완벽히 지원하게 되었습니다.
