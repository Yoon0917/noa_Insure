# 신규 기능 구현 기록: 장애/미처리 전용 대기열 (DLQ)

## 구현 일자: 2026-04-25

### 📌 기획 배경 및 목적

관제 플랫폼에서 에러가 빈번히 발생할 경우, 메인 대시보드에서 건별로 재처리하는 방식은 운영자에게 피로도를 줍니다. 이를 해결하기 위해 실패한 통신 건들만 따로 격리하여 집중적으로 모니터링하고 **일괄(Bulk) 처리**할 수 있는 전용 큐(Dead Letter Queue) 시스템이 필요했습니다.

### ⚙️ 핵심 구현 사항

1. **SPA 기반 독립된 메뉴 (Routing)**
   - 좌측 LNB에서 "장애/미처리 대기열" 클릭 시, 메인 대시보드가 `hidden` 처리되고 전용 뷰가 나타나도록 바닐라 자바스크립트 기반의 가벼운 라우팅을 구현했습니다.

2. **에러 사유별 필터링 (Filter by Status Code)**
   - `500 (Internal Error)`, `504 (Timeout)`, `401 (인증 실패)` 등 에러 응답 코드별로 뷰를 즉시 좁혀 원인별 집중 조치가 가능하도록 드롭다운 필터를 적용했습니다.

3. **일괄 재처리 (Bulk Retry) 및 UX 최적화**
   - 다중 선택(Checkboxes) 및 전체 선택 기능을 추가했습니다. 여러 건을 선택 후 버튼 클릭 시 비동기(`Promise.all`)로 동시에 재처리 API가 호출되며, 완료 후 토스트 피드백이 제공됩니다.

4. **수동 상태 변경 (Discard)**
   - 복구가 불가능하거나 잘못된 데이터로 인한 영구 실패 건을 관리자가 수동으로 '처리 포기(DISCARDED)' 상태로 전환할 수 있습니다.
   - 이 상태로 전환된 건들은 메인 대시보드의 '전체/장애 통계' 데이터 및 리스트에서 완전히 소거되어 운영자의 관제 리소스 낭비를 막아줍니다.
